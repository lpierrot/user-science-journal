Accessibility

Posted by Lionel Pierrot on May 12, 2019

For this week's accessibility report, I tested this blog to see if it adheres W3C principles and Section 508 guidelines. During this test, I used muliple tools to check for errors and alternative ways of fixing them.

As part of a class project and familiarizing myself with accessibility best practices, I will document any errors and fix them as I continue to add more content. This blog is not targeting any particular demography, rather, anyone who enjoys the web and interested in learning more about accessibility.

For testing purposes, I used. Google's lightweight, VoiceOver by Apple, WAVE and WCAG 2.0 standards for conformance evaluation. I also used the Webaim checklist to help me refer back to W3C principles.

Blog homepage detailed summary

According to WAVE audit, there nine errors on the homepage alone. Most errors are empty links in the footer and the drop-down menu. I also noticed contrast errors but were good enough to pass the test. Looking at the code below it easy to fix these errors. I can just go back to my code editor and remove those extra lines of code or the actual pages.
